token boundary. For example: the system is is unstable under load,

The fix is is simple: dedupe at the post-processor layer. But this
detector merely flags; it does not repair, because repair belongs in
a separate concern.

Inside a fenced block, repeated words like `the the` should NOT be
flagged because code is code:

```python

```

But here in prose, "the the" should be caught. Same for and and
again. A trailing duplicate at end of line should also fire fire.
