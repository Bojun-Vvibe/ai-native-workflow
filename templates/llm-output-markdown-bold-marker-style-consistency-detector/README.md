# llm-output-markdown-bold-marker-style-consistency-detector

Detects mixed bold-emphasis marker styles within a single Markdown document.

CommonMark accepts both `**bold**` and `__bold__`. Either is fine on its
own — but mixing them in one document is a strong signal the text was
stitched together from multiple sources, or generated by an LLM that
sampled inconsistent training data.

## Why this matters

* Many static-site renderers normalize the markers, hiding the drift in
  rendered output but leaving it in source diffs and `git blame`.
* Some Markdown processors (older Pandoc dialects, MultiMarkdown) treat
  `__` as something else entirely, producing different rendered HTML.
* Style-guide enforcement: most house styles pick one and stick with it.

## How it decides

1. Scan the document for bold runs of either style, ignoring fenced code
   blocks and inline `code` spans.
2. Whichever style appears MORE often becomes the de-facto canonical for
   this document. Ties are treated as canonical = `**`.
3. Every occurrence of the OTHER style is flagged with line, column, the
   matched text, and the canonical style.

If only one style appears (or neither), the document is clean.

## Edge cases handled

* **Fenced code blocks** (` ``` ` and ` ~~~ `) — entire blocks skipped.
* **Inline code spans** — replaced with whitespace before scanning so
  embedded `**` / `__` inside `` `code` `` does not count.
* **Intra-word `__`** — CommonMark forbids intra-word `_` emphasis, so
  the regex requires non-word characters (or string boundary) on the
  outer side of each delimiter run. Prevents false positives like
  `var__name__` in prose discussing Python dunder methods.

## Edge cases NOT handled (yet)

* Triple emphasis (`***bold italic***` / `___bold italic___`) — passes
  through as italic-of-bold; both forms accepted.
* Multi-line bold runs — only single-line matches are considered.
* Cross-file consistency — single-file only.

## How to run

```bash
python3 detect.py example/bad.md
```

Exit codes:

* `0` — clean (single style, or no bold)
* `1` — mixed-style findings printed to stdout
* `2` — usage / IO error

## CI usage

```yaml
- name: Lint markdown bold-marker style
  run: |
    find docs -name '*.md' -print0 | \
      xargs -0 -n1 python3 templates/llm-output-markdown-bold-marker-style-consistency-detector/detect.py
```

## Worked example

`example/bad.md` uses `**bold**` four times and `__bold__` twice, so the
canonical style is resolved to `**bold**` and both `__bold__` runs are
flagged. Inline code and fenced blocks containing literal `**` are
correctly ignored. Output matches `example/expected-output.txt` exactly
and the script exits `1`.
