# Good Document

All link titles use double-quoted style consistently.

See the [spec page](https://example.org/spec "Specification overview")
and the [reference page](https://example.org/ref "Reference index")
plus the [FAQ](https://example.org/faq "Frequently asked questions").

Links without titles are fine: visit [home](https://example.org/)
or [about](https://example.org/about).

Code spans like `[fake](url "x")` inside backticks are correctly ignored.

```
[also fake](url 'inside-fence')
```
